Git e GitHub

O que é Git?

Git é um programa que guarda cópias do seu trabalho no computador.

Se você errar, pode voltar para uma versão antiga.

Ajuda a não perder nada do que você fez.

Git x GitHub

Git GitHub
Programa instalado no computador Site na internet
Guarda cópias do seu trabalho Guarda seu trabalho online
Ajuda a não perder nada Permite acessar de qualquer lugar
Funciona mesmo sem internet Precisa de internet para acessar
Você salva (commit) no seu PC Você envia (push) para o site

Git é onde você trabalha. GitHub é onde você compartilha.

Por que usar?

  • Não perde o que fez
  • Pode voltar atrás se errar
  • Pode trabalhar junto com amigos
  • Seu trabalho fica salvo na internet

Como funciona?

  1. Você faz uma mudança no seu trabalho
  2. Salva essa mudança (isso se chama "commit")
  3. Envia para o GitHub (isso se chama "push")
  4. Pode pegar mudanças novas do GitHub (isso se chama "pull")

Palavras importantes

  • Commit: Salvar o que fez
  • Push: Enviar para o GitHub
  • Pull: Pegar do GitHub
  • Branch: Linha separada para testar coisas
  • Merge: Juntar as linhas

Passos simples

  1. Fazer um projeto
  2. Fazer mudanças
  3. Salvar (commit)
  4. Enviar para o GitHub (push)

Está­gios dos arquivos no Git

O que significam os estágios?

  • Untracked: Arquivo novo, o Git ainda não está cuidando dele.
  • Modified: Arquivo já conhecido pelo Git, mas foi alterado e ainda não está pronto para salvar.
  • Staged: Arquivo pronto para ser salvo (commit), já foi adicionado com git add.
  • Unmodified: Arquivo já salvo, sem mudanças desde o último commit.

O ciclo normal é: Untracked → Staged → Committed (Unmodified). Se mudar de novo, volta para Modified.

Comandos mais usados


git init      (começa a usar o Git)
git add .     (prepara tudo para salvar)
git commit -m "mensagem" (salva)
git push      (envia para o GitHub)
git pull      (pega do GitHub)

Vamos praticar!

Vamos tentar juntos no computador.

Atividade prática: Subindo um site no GitHub

  1. Crie um arquivo HTML simples
    Exemplo: index.html com <h1>Olá, GitHub!</h1>
  2. Abra o terminal na pasta do arquivo
  3. Inicie o Git
    git init
  4. Adicione o arquivo para controle do Git
    git add index.html
  5. Salve a mudança (commit)
    git commit -m "meu primeiro site"
  6. Crie um repositório novo no GitHub
    (No site do GitHub, clique em New e siga os passos)
  7. Ligue seu projeto local ao GitHub
    git remote add origin https://github.com/SEU_USUARIO/NOME_DO_REPO.git
  8. Envie seu arquivo para o GitHub
    git push -u origin main
    (ou git push -u origin master se for esse o nome do branch)
  9. Veja seu arquivo no GitHub!

Trabalhando em grupo

Cada pessoa pode ter uma cópia do projeto.

Todos podem fazer mudanças e depois juntar tudo.

Assim, ninguém perde o trabalho do outro.

O que é uma Branch?

É como uma estrada separada para testar ideias.

Se der certo, junta com o projeto principal.

Se não der, não atrapalha o resto.

Visualizando uma Branch

Imagine cada branch como um caminho diferente no seu projeto. Você pode criar, mudar e juntar esses caminhos!

git status

Mostra o que mudou e o que falta salvar.

git status

Exemplo: Veja se há arquivos novos ou modificados antes de salvar.

git log

Mostra a lista de tudo que já foi salvo (commits).

git log

Exemplo: Veja o histórico de mudanças do projeto.

git diff

Mostra as diferenças entre arquivos ou versões.

git diff

Exemplo: Veja o que mudou antes de salvar (commit).

git checkout

Troca de branch ou recupera arquivo antigo.

git checkout nome-da-branch

Exemplo: Mude para outro "galho":
git checkout main
git checkout {nome-da-branch} 

git checkout -b nome-da-branch

Exemplo: Crie e Mude para outra "galho"

git branch

Mostra ou cria novas linhas de trabalho (branches).

git branch

Exemplo: Veja todas as branches do projeto.

git branch nova-ideia

Exemplo: Crie uma nova branch chamada "nova-ideia".

git rm

Remove arquivos do controle do Git.

git rm arquivo.txt

Exemplo: Remova o arquivo "arquivo.txt" do projeto.

git reset

Desfaz mudanças que ainda não foram salvas.

git reset arquivo.txt

Exemplo: Desfaça mudanças feitas no arquivo "arquivo.txt" antes de salvar (commit).
